В данной задачи использовался паттер стратегия для реализации различных вариантов валидации пароля.
Также, для комбинации различных вариантов проверки реализован класс CompositeValidator.cs.
Также был создан интерфейс для репозитория, чтобы в дальнейшем можно было использовать другой репозиторий нежели SQL и код клиент не пострадал от этого.
